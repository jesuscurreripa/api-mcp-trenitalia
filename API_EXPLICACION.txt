API TRENITALIA PALERMO - EXPLICACION

Objetivo
Esta API local sirve como proxy controlado hacia las APIs web de Trenitalia/lefrecce.it. La idea es que una app o un agente AI no llame directamente desde el navegador a Trenitalia, porque esas APIs no exponen CORS abierto para frontend. En cambio, este backend FastAPI hace la llamada desde servidor y devuelve una respuesta mas simple.

Base local
http://127.0.0.1:8000

Documentacion interactiva
http://127.0.0.1:8000/docs

Archivos principales
app/main.py
Define la aplicacion FastAPI, CORS local y los endpoints HTTP.

app/stations.py
Contiene la lista cerrada de estaciones que nos interesan. Esta lista limita la API a Palermo-Punta Raisi, Palermo-Cefalu, Catania Centrale y Agrigento Centrale.

app/trenitalia.py
Construye el payload que espera lefrecce.it, llama a /ticket/solutions y resume cada solucion con horario, precio, estado y trenes.

estaciones_interes.txt
Lista humana de estaciones e IDs Trenitalia permitidos.

requirements.txt
Dependencias Python del proyecto.

Endpoints

GET /health
Comprueba que el servidor esta vivo.
Respuesta:
{"status":"ok"}

GET /stations
Devuelve todas las estaciones permitidas.
Parametros opcionales:
q: texto para buscar por nombre o alias.
line: una de estas opciones:
- palermo_punta_raisi
- palermo_cefalu
- extra

Ejemplos:
GET /stations
GET /stations?q=aeroporto
GET /stations?line=palermo_cefalu

GET /stations/{station_id}
Devuelve una estacion concreta si el ID esta en la lista permitida.
Ejemplo:
GET /stations/830012133

GET /lines
Devuelve las estaciones agrupadas por linea:
- palermo_punta_raisi
- palermo_cefalu
- extra

POST /solutions
Busca trenes entre dos estaciones permitidas.

Body JSON:
{
  "fromStationId": 830012002,
  "toStationId": 830012133,
  "departureTime": "2026-05-06T10:30:00+02:00",
  "adults": 1,
  "children": 0,
  "criteria": {
    "frecceOnly": false,
    "regionalOnly": true,
    "noChanges": false,
    "order": "DEPARTURE_DATE",
    "limit": 10,
    "offset": 0
  }
}

Campos importantes:
fromStationId: ID Trenitalia de origen. Debe estar en estaciones_interes.txt.
toStationId: ID Trenitalia de destino. Debe estar en estaciones_interes.txt.
departureTime: fecha/hora ISO con zona horaria, preferiblemente Europe/Rome, por ejemplo 2026-05-06T10:30:00+02:00.
adults: adultos, minimo 1.
children: ninos, minimo 0.
regionalOnly: para las rutas locales conviene true.
noChanges: true si solo quieres trenes directos.
limit: cantidad maxima de soluciones, entre 1 y 50.
offset: desplazamiento para pedir mas resultados.

Respuesta resumida:
searchId: ID de busqueda devuelto por Trenitalia.
cartId: ID de carrito devuelto por Trenitalia.
count: numero de soluciones.
solutions: lista resumida con origen, destino, salida, llegada, duracion, estado, precio y trenes.
raw: respuesta original completa de Trenitalia por si hace falta depurar o usar campos avanzados.

POST /day-solutions
Busca todas las soluciones de un dia desde una hora indicada en adelante. Este endpoint resuelve automaticamente la paginacion con offset para que la persona no tenga que pedir offset 0, 10, 20, etc.

Regla de uso conversacional:
Si el usuario pide "los trenes de hoy" o "busca para hoy" pero no dice hora de inicio, el agente debe preguntar:
"A partir de que horario quieres la busqueda?"

Ejemplo:
Si el usuario dice "desde las 6 de la tarde", usar:
"startTime": "18:00"

Body JSON:
{
  "fromStationId": 830012002,
  "toStationId": 830012133,
  "date": "2026-05-06",
  "startTime": "18:00",
  "adults": 1,
  "children": 0,
  "regionalOnly": true,
  "noChanges": false,
  "pageSize": 10,
  "maxPages": 12
}

Respuesta:
from: fecha/hora exacta desde la que se busco.
date: dia filtrado.
pageSize: tamano usado por pagina.
pagesFetched: cantidad de paginas consultadas a Trenitalia.
count: total de soluciones devueltas.
solutions: trenes del mismo dia con salida igual o posterior a startTime.
fromStation: estacion origen validada.
toStation: estacion destino validada.

Ejemplo esperado:
Con startTime 18:00 en Palermo Centrale -> Palermo Aeroporto, la primera salida debe ser 18:12 si Trenitalia mantiene el horario actual.

Notas operativas
La API valida que origen y destino esten dentro de nuestra lista curada. Esto evita que una app o agente consulte estaciones fuera del alcance definido.
Los datos de horarios/precios vienen en vivo desde Trenitalia y pueden cambiar.
El endpoint externo usado es:
https://www.lefrecce.it/Channels.Website.BFF.WEB/website/ticket/solutions
